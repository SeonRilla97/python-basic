1. 기계 중심 : 기계에 명령
2. 절차 지향 : 명령어의 순차적 실행
3. 구조적 프로그래밍 : 함수 중심 프로그램 절차적 구성

4. 객체 지향 : 데이터 + 기능을 하나로 묶어서 관리 (객체화) - 객체간 상호작용
   - 데이터와 행위가 하나의 객체로 캡슐화

===

Class VS Object

    개체 : 객체
    상태 : 데이터
    행위 : 메서드

Class : 틀
Object : 틀로 만든 실체

Responsibility : 자기 일은 자기가 책임진다.
Modeling : 추상화를 통해 문제를 해결한다.
Custom Type : 나만의 자료형을 만든다.
Message Passing : 객체간의 상호작용 (메시지를 보낸다)

===

객체의 관계

is-a : ~은~의 일종이다. (상속)

- 학생은 사람의 일종이다.
- 개는 동물의 일종이다.
  같은 분류인가? 부모타입으로 완전 대체 가능한가?

has-a : ~은~을 가지고 있다. (포함)

- 경찰은 총을 가지고 있다.
- 차는 엔진을 가지고 있다.
  내 기능은 내가 직접하지 않고 다른 객체에게 위임한다.

Composition : 부품과 전체의 생명주기가 일치 (사람&심장)
Aggregation : 부품과 전체의 생명주기가 독립적 (학교&학생)

★★★★ 상속 보다는 포함을 우선할것!★★★★

- 상속 : 부모 구현 의존, 부모 변경등의 유연성 불가, 클래스 폭발 => 분류를 표현할 때 사용
- 포함 : 블랙박스 재사용, 런타임 교체, 단순성 => 역할 조립, 유연한 변경
